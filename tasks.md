# Linux
1) Запустить приложение [crypto-app](https://github.com/AnastasiyaGapochkina01/starkjjj/tree/main/crypto-app) как systemd сервис
**Требования**
   - приложение запущено от имени пользователя crypto
   - директория с данными БД вынесена на отдельный диск с файловой системой xfs
   - перед приложением стоит nginx в роли reverse proxy

2) Запустить приложение [catalog-app](https://github.com/AnastasiyaGapochkina01/starkjjj/tree/main/catalog-app)как systemd сервис
**Требования**
   - приложение запущено от имени пользователя shop
   - директория с данными БД вынесена на отдельный диск с файловой системой xfs
   - перед приложением стоит nginx в роли reverse proxy
**Подзадача**
Приложение периодически тупит. Надо выяснить причину.

3) Для обоих проектов написать скрипты резервного копирования с проверкой валидности и синхронизацией бэкапов на _удаленный_ хост.

4) Написать скрипт для синхронизации БД `prod -> dev` для приложения catalog-app; скрипт должен при синхронизации удалять таблицу payments.

5) Создать несколько cron-задач по мониторингу системы и написать скрипт для их включения и выключения

6) Реализовать сценарий переезда приложения crypto-app в другое облако/ЦОД

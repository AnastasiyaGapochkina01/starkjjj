# Каталог товаров + PostgreSQL + сервис обновления остатков

Проект поднимает 3 контейнера:
- `postgres` — база данных PostgreSQL.
- `app` — Go-приложение с HTML-каталогом и JSON API.
- `stock-updater` — отдельный Python-сервис, который по cron-расписанию обновляет остатки.

## Что умеет
- Показывает каталог товаров из PostgreSQL.
- Отдает JSON по `GET /api/products`.
- Инициализирует БД схемой и тестовыми товарами.
- Отдельный updater обновляет остатки по расписанию из переменной окружения `SCHEDULE`.


После старта:
- Каталог: `http://localhost:8080`
- API: `http://localhost:8080/api/products`

## Ключевые переменные updater
- `SCHEDULE` — cron, например `*/1 * * * *`
- `HANG_PATTERN` — шаблон зависаний, по умолчанию `1,1,1,0,0` (3 из 5 запусков зависают)
- `HANG_SECONDS` — сколько секунд висеть
- `CPU_BURN_SECONDS` — сколько секунд грузить CPU перед обычным sleep
- `BATCH_SIZE` — сколько товаров обновлять за запуск
- `STOCK_MIN_DELTA`, `STOCK_MAX_DELTA` — диапазон изменения остатков


## Структура
- `db/init.sql` — схема и сиды
- `app/` — веб-приложение на Go
- `updater/` — фоновый сервис на Python
